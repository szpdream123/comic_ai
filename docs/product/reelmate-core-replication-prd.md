# ReelMate / 万兴剧厂核心体验复刻 PRD

> 版本：v2  
> 目标：把公开竞品文档中的核心体验拆成可设计、可研发、可测试的 P0 复刻规格。  
> 依据：`docs/product/reelmate-create-guide.raw.md`。  
> 边界：本文只写用户体验、功能、状态、规则、验收。AIComicBuilder、Kernel、资产同步、模型网关等实现约束移到 `docs/product/reelmate-technical-integration-notes.md`。

## 0. 证据分层

本文每条关键需求按三层标注：

- **A 竞品已确认事实**：公开文档明确写出。
- **B 高置信推导**：由竞品事实和 AI 漫剧生产流程直接推出，但未实机验证。
- **C 我方设计决策**：为落地 MVP、B 端化或长期架构做出的我方选择，不等同于竞品事实。

## 1. 复刻目标与范围

### 1.1 复刻对象

| 项 | 内容 | 来源 | 置信度 |
| --- | --- | --- | --- |
| 竞品 | ReelMate / 万兴剧厂 | A | 高 |
| 参考文档 | 万兴剧厂 - 新手指引 | A | 高 |
| 复刻端 | Web 桌面端 | A：文档说明需电脑设备访问，移动设备当前暂不支持功能使用 | 高 |
| 复刻层级 | 核心功能结果、用户路径、关键交互、业务规则；不做像素级视觉复刻 | C | 高 |

### 1.2 P0 复刻范围

P0 只保留完成“剧本到分镜图/分镜视频/素材导出”的最小闭环：

1. 创建项目；
2. 上传或粘贴标准剧本；
3. 剧本解析；
4. 角色、场景、道具资产提取；
5. 资产确认与编辑；
6. 分镜拆解；
7. 风格校准；
8. 单个/批量生成分镜图；
9. 单个分镜编辑与重生成；
10. 单个分镜图转视频；
11. 任务状态展示；
12. 基础额度校验；
13. 导出素材包。

### 1.3 P1/P2/P3 范围

| 优先级 | 能力 | 处理方式 |
| --- | --- | --- |
| P1 | 批量视频生成、视频合成、项目消耗明细、失败任务聚合 | 影响效率，核心闭环可先不依赖 |
| P1 | 完整生成历史列表 | 便于追溯，但 P0 只要求不覆盖旧结果 |
| P2 | 工具箱、成员邀请、角色细权限、站内通知、版本回滚、版本对比、版本备注 | 增强体验 |
| P3 | 小说库、剧本库、复杂付费套餐、客户审片、移动端、完整审核流 | 后置 |

### 1.4 不复刻

- 不复刻竞品视觉样式的像素级细节；
- 不复刻竞品底层技术架构；
- 不在 P0 复刻小说库、剧本库；
- 不在 P0 复刻移动端；
- 不在 P0 复刻完整会员/付费套餐规则。

## 2. P0 核心体验指标

这些指标是我方设计决策，用来让“顺畅”可验收。

| 指标 | P0 目标 | 来源 |
| --- | --- | --- |
| 核心链路阶段数 | 从创建项目到首批分镜图完成，核心阶段不超过 12 个 | C |
| 用户主动关键决策点 | 从创建项目到首批分镜图完成，关键决策点不超过 8 个；批量确认按 1 个决策点计算 | C |
| 剧本解析启动反馈 | 点击解析后 1 秒内进入 loading / queued 状态 | C |
| 长任务反馈 | 所有超过 5 秒的任务必须展示进度或阶段状态 | C |
| 单任务失败恢复 | 用户从失败态到重新发起任务不超过 3 步 | C |
| 批量分镜部分成功 | 单个分镜失败不影响其他分镜完成 | B/C |
| 资产校准 | 批量生成前至少完成 3 张代表性校准图或显式跳过确认 | A/C |
| 导出前检查 | 缺失资产时明确列出缺失项，不允许静默导出失败 | C |

## 3. 用户核心路径

### 3.1 主路径：标准剧本到素材包

| 步骤 | 用户看到 | 用户操作 | 系统反馈 | 状态变化 | 来源 |
| --- | --- | --- | --- | --- | --- |
| 1 | 项目入口 | 点击创建项目 | 打开创建表单 | `none → project_draft` | B/C |
| 2 | 项目创建表单 | 填项目名、画幅、分辨率，上传/粘贴剧本 | 创建成功进入项目工作台 | `project_draft → script_input` | B/C |
| 3 | 剧本模块 | 点击解析剧本 | 显示解析中；完成后展示剧集和资产候选 | `script_input → parsing → assets_pending` | A/B |
| 4 | 公共资产模块 | 检查角色、场景、道具，编辑/确认 | 资产卡片进入已确认或需修正 | `assets_pending → assets_reviewing` | A/B |
| 5 | 分镜模块 | 点击拆解分镜 | 生成分镜列表 | `assets_reviewing → shots_ready` | A/B |
| 6 | 风格校准模块 | 从已拆解分镜中选择 3 张代表分镜并生成校准图 | 显示校准结果 | `shots_ready → calibration_pending` | A |
| 7 | 风格校准模块 | 点击校准通过 | 解锁分镜批量生成 | `calibration_pending → calibration_passed` | A/C |
| 8 | 分镜模块 | 点击批量生成分镜图 | 分镜逐个进入生成中并回填结果 | `calibration_passed → image_generating → image_completed / image_failed` | A/B |
| 9 | 分镜详情 | 对不满意分镜编辑并重生成 | 生成新结果，不覆盖旧结果 | `image_completed → image_stale → image_generating → image_completed` | A/B/C |
| 10 | 分镜详情 | 点击图转视频 | 显示视频生成中，完成后可预览 | `video_ready → video_generating → video_completed / video_failed` | A/B |
| 11 | 导出模块 | 点击导出素材包 | 检查资产完整性，生成下载链接 | `export_preparing → export_ready` | A/B/C |

### 3.2 分支路径

| 分支 | 触发条件 | 用户路径 | 来源 |
| --- | --- | --- | --- |
| 已有资产 | 用户已有角色/场景/道具参考图 | 在公共资产模块上传参考图，跳过单项生成 | B |
| 校准失败 | 3 张校准图风格/角色不一致 | 返回公共资产或提示词设置，修改后重新校准 | A |
| 单镜失败 | 某个分镜图生成失败 | 打开失败分镜，修改描述/参考图后重试 | A/B |
| 主体过小 | 视频生成主体几乎不动 | 重新生成更近首帧，人物占画面 1/3 到 2/3 | A |
| 人物过多 | 多人乱动或主角错位 | 拆分分镜，一个分镜一个主要运动人物 | A |
| 额度不足 | 任务预估成本超过可用额度 | 阻止任务，提示额度不足和降低范围 | C |

### 3.3 退出路径

| 退出点 | 规则 | 来源 |
| --- | --- | --- |
| 任意模块返回项目列表 | 已保存数据保留，后台任务继续执行 | C |
| 任务执行中关闭页面 | 任务不应被中断，返回后显示最新状态 | C |
| 项目未完成 | 项目列表显示当前阶段和阻塞项 | C |

## 4. 页面与模块规格

### 4.1 项目创建

| 项 | 规格 |
| --- | --- |
| 页面目标 | 创建一个可进入生产流程的项目 |
| 入口 | 项目列表主 CTA |
| 首屏结构 | 项目基础信息表单 + 剧本输入区 + 创建按钮 |
| 默认状态 | 空表单，主 CTA 禁用，直到项目名和剧本输入满足要求 |
| 主 CTA | 创建并解析 |
| 次级操作 | 仅创建项目、取消 |
| 成功跳转 | 项目工作台的剧本/解析进度模块 |
| 失败停留 | 当前表单，字段级错误优先 |

字段：

| 字段 | 必填 | 规则 | 来源 |
| --- | --- | --- | --- |
| 项目名称 | 是 | 1-60 字；同工作区可重名但不推荐 | C |
| 剧本输入 | 是 | 文件上传或文本粘贴二选一 | A/B |
| 画幅比例 | 是 | 9:16 或 16:9；默认 9:16 | A/C |
| 分辨率 | 是 | 720P 或 1080P；默认 1080P | A/C |
| 项目预算 | 否 | 正数；不填则仅受组织额度约束 | C |
| 视觉风格 | 否 | 文本或预设，P0 可用文本 | A/B/C |

### 4.2 项目工作台

| 项 | 规格 |
| --- | --- |
| 页面目标 | 告诉用户项目当前卡在哪一步，并提供下一步主操作 |
| 默认进入模块 | 根据项目状态进入最早未完成模块 |
| 首屏结构 | 顶部项目状态条 + 阶段导航 + 当前模块 + 右侧任务摘要 |
| 主 CTA | 随状态变化，如解析剧本、确认资产、开始校准、拆解分镜、批量生成 |
| 状态驱动变化 | 每个阶段只突出一个主 CTA，阻塞项显示在 CTA 上方 |
| 失败聚合 | 顶部状态条显示失败数，点击跳到失败项列表 |
| 结果回填 | 任务完成后在对应模块原位展示结果 |

默认模块规则：

| 项目状态 | 默认模块 | 主 CTA |
| --- | --- | --- |
| `project_draft` | 剧本 | 上传/粘贴剧本 |
| `script_input` | 剧本 | 解析剧本 |
| `parsing` | 剧本 | 查看解析进度 |
| `assets_pending` | 公共资产 | 确认/生成公共资产 |
| `assets_reviewing` | 分镜 | 拆解分镜 |
| `shots_ready` | 风格校准 | 选择代表分镜并生成校准图 |
| `calibration_pending` | 风格校准 | 查看/重生成校准图 |
| `calibration_passed` | 分镜 | 批量生成分镜图 |
| `image_partial_failed` | 分镜 | 修复失败分镜 |
| `exportable` | 导出 | 导出素材包 |

### 4.3 剧本模块

| 项 | 规格 |
| --- | --- |
| 页面目标 | 完成剧本输入、解析和剧集/资产候选生成 |
| 默认状态 | 无剧本时显示上传和粘贴入口；有剧本时显示文本预览 |
| 主 CTA | 解析剧本 |
| 次级操作 | 替换剧本、保存文本、清空 |
| 加载态 | 展示阶段：上传中、解析中、生成候选中 |
| 成功态 | 展示剧集结构、角色/场景/道具候选摘要 |
| 失败态 | 显示失败原因，提供重试和手动编辑 |

输入：

- 文件：P0 支持 `.txt`、`.docx`。`.pdf` 标记为 P1，除非后续验证其接入成本和解析质量都满足 P0 要求。
- 文本：不少于 100 字；超过上限时提示分段或上传文件。具体上限由实现压测决定，但 PRD 要求前端展示明确限制。

输出：

- 原始剧本资产；
- 剧集列表；
- 角色候选；
- 场景候选；
- 道具候选；
- 解析任务记录。

### 4.4 公共资产模块

| 项 | 规格 |
| --- | --- |
| 页面目标 | 让用户确认进入分镜生成前的角色、场景、道具资产 |
| 首屏结构 | 角色/场景/道具三个 Tab；每个 Tab 为资产卡片列表 |
| 默认状态 | 展示提取候选；未提取时主 CTA 为提取资产 |
| 主 CTA | 确认公共资产 |
| 次级操作 | 编辑描述、生成参考图、上传参考图、重新生成、删除候选 |
| 阻塞态 | 存在未确认关键角色或主要场景时，不能进入校准通过 |

资产卡片字段：

| 字段 | 说明 | 来源 |
| --- | --- | --- |
| 名称 | 角色/场景/道具名称 | A/B |
| 类型 | 角色、场景、道具 | A |
| 描述 | 可编辑文本 | A/B |
| 参考图 | 上传或生成 | A/B |
| 状态 | 待确认、生成中、已确认、失败、需修正 | C |
| 使用范围 | 项目/剧集，P0 默认项目级 | C |

关键资产定义：

| 类型 | P0 定义 | 系统推荐规则 | 用户调整 | 来源 |
| --- | --- | --- | --- | --- |
| 关键角色 | 对主线叙事或高频分镜有影响的角色 | 角色在剧本文本中出现次数 Top 5；或拥有对白；或出现在前 20% 剧情中 | 用户可手动标记/取消关键角色 | B/C |
| 主要场景 | 承载多镜头或关键剧情的场景 | 场景出现次数 Top 5；或关联分镜数量超过 3；或出现在开场/转折/高潮片段 | 用户可手动标记/取消主要场景 | B/C |
| 关键道具 | 影响剧情推进或视觉识别的道具 | 被多个分镜引用；或出现在分镜核心信息点中 | 用户可手动标记/取消关键道具 | B/C |

阻塞规则：

- 关键角色和主要场景必须确认后，才允许通过风格校准。
- 关键道具未确认时不阻塞校准通过，但在相关分镜生成前提示。
- 系统推荐只是默认值，用户最终可以手动调整关键资产标记。

### 4.5 风格校准模块

| 项 | 规格 |
| --- | --- |
| 页面目标 | 在完整分镜拆解后，批量生成前验证角色一致性、画风、色彩、光影、线条表现 |
| 首屏结构 | 校准说明 + 3 个校准分镜槽位 + 生成结果区 |
| 默认状态 | 未选择校准分镜时主 CTA 禁用 |
| 主 CTA | 生成校准图 |
| 次级操作 | 替换校准分镜、重新生成、通过校准、标记不通过 |
| 成功态 | 用户可点击通过校准，解锁批量生成 |
| 失败态 | 展示失败分镜和重试入口 |

校准规则：

- 需要从已拆解的分镜列表中选择 3 张代表性分镜：典型中景、近景/特写、情绪/氛围镜头。来源 A/C。
- 校准通过后再放量。来源 A。
- 校准不通过时停止批量生成，优先修正风格参数与参考资产。来源 A。
- P0 允许管理员或项目负责人显式跳过校准，但需要二次确认并记录审计。来源 C。

### 4.8 分镜视频模块

| 项 | 规格 |
| --- | --- |
| 页面目标 | 将单个分镜图转为视频片段 |
| 入口 | 分镜详情或分镜列表操作 |
| 默认状态 | 无分镜图时禁用图转视频 |
| 主 CTA | 生成视频 |
| 次级操作 | 编辑视频提示词、重新生成、下载片段 |
| 失败态 | 按失败类型给修复建议 |

视频提示词规则：

- 镜头不够精彩：补充镜头运动，如 dolly in、dolly out、pan、follow shot。来源 A。
- 人物运动不对：明确谁动、怎么动、往哪动。来源 A。
- 人物太小：重新生成首帧，人物占画面 1/3 到 2/3。来源 A。
- 人物过多：拆分分镜，一个分镜一个主要运动人物。来源 A。

## 5. P0 功能规则

### 5.1 创建项目

| 项 | 规则 |
| --- | --- |
| 触发 | 项目列表点击创建项目 |
| 输入 | 项目名称、剧本、画幅、分辨率、项目预算可选、视觉风格可选 |
| 校验 | 项目名称必填；剧本文件或文本必填；画幅/分辨率必选 |
| 输出 | 项目记录、剧本资产、解析任务 |
| 成功 | 进入项目工作台，项目状态为 `script_input` 或 `parsing` |
| 失败 | 停留创建页，字段错误优先展示 |
| 取消 | 创建前可取消；创建后不可取消项目创建，只能删除/归档，P0 可后置 |

### 5.2 解析剧本

| 项 | 规则 |

### 5.10 P0 额度校验与占用

P0 总体覆盖生成前预估额度校验、额度占用和成功后消耗记录；其中完整额度占用/释放/对账从 P0-B 开始。不做复杂退款、多退少补、后付费账单和供应商差额结算。

架构分期口径：

- P0-A Internal Alpha：同步余额校验 + 单任务幂等防重；不承诺强并发额度正确性，不做完整预留/释放/对账。
- P0-B Commercial Beta Gate：创建生成类工作流时，在同一数据库事务内完成额度校验、预留和任务级分配；任务成功/失败/取消/unknown 按分配粒度结算。
- P0-C Commercial Ops Hardening：接入供应商实际成本回填和成本对账。

| 项 | 规则 |
| --- | --- |
| 校验时机 | 用户点击生成类任务前校验，包括资产参考图、校准图、批量分镜图、图转视频、导出包 |
| 校验范围 | 按当前任务范围估算总消耗；批量任务按待生成子任务数量估算 |
| 占用时机 | P0-A 为同步余额校验；P0-B 起创建生成类任务时，在同一数据库事务内完成额度校验和额度占用 |
| 额度不足 | 不创建任务、不占用额度，停留当前页面，提示“额度不足，无法开始生成” |
| 项目预算不足 | 不创建任务、不占用预算，提示项目预算不足；用户可减少生成范围或调整预算 |
| 任务成功 | 将占用额度转为实际消耗；P0-A/P0-B 可用估算值作为实际消耗，待 P0-C 接入供应商成本回填 |
| 任务失败 | P0-B 默认释放占用额度，记录 0 成本失败事件；若供应商已产生异常成本，仅记录内部异常成本，不向用户扣减 |
| queued 取消 | P0-B 取消成功后释放占用额度 |
| running 取消 | P0 不承诺取消成功；P0-B 中若最终失败按失败规则释放占用，若最终成功按成功规则转为消耗 |
| 重复提交 | 同一任务运行中不允许重复创建；按钮禁用，刷新后仍能恢复运行中状态 |

后续阶段再补：

- 复杂冻结退款；
- 供应商实际成本回填（P0-C）；
- 多退少补；
- 供应商已扣费但平台任务失败的异常成本处理；
- 后付费账单。

## 6. 状态机

### 6.1 项目状态

```text
project_draft
  → script_input
  → parsing
  → assets_pending
  → assets_reviewing
  → shots_ready
  → calibration_pending
  → calibration_passed
  → image_generating
  → image_partial_failed
  → exportable
```

失败不是单一终态。P0 使用局部失败状态，不把整个项目轻易置为 `failed`。

| 状态 | 来源 | 可编辑 | 阻塞下一步 | 允许导出 | 可流转到 |
| --- | --- | --- | --- | --- | --- |
| `project_draft` | 创建未完成 | 是 | 是 | 否 | `script_input` |
| `script_input` | 已有剧本 | 是 | 是 | 否 | `parsing` |
| `parsing` | 解析中 | 否，避免输入变更 | 是 | 否 | `assets_pending`、`script_input` |
| `assets_pending` | 资产候选生成 | 是 | 是 | 否 | `assets_reviewing` |
| `assets_reviewing` | 用户确认资产 | 是 | 是，阻塞校准通过 | 否 | `shots_ready` |
| `shots_ready` | 分镜已生成，可选择校准分镜 | 是 | 是，阻塞批量生成 | 否 | `calibration_pending` |
| `calibration_pending` | 校准图生成中或待确认 | 是 | 是，阻塞批量生成 | 否 | `calibration_passed`、`shots_ready` |
| `calibration_passed` | 校准已通过或已显式跳过 | 是 | 否 | 否 | `image_generating` |
| `image_generating` | 分镜图生成中 | 部分可编辑，运行中项不可编辑 | 否 | 部分允许 | `exportable`、`image_partial_failed` |
| `image_partial_failed` | 部分分镜失败 | 是 | 否 | 部分允许 | `image_generating`、`exportable` |
| `exportable` | 有可导出资产 | 是 | 否 | 是 | `image_generating` |

### 6.2 分镜状态

```text
draft → ready → image_generating → image_completed → video_ready → video_generating → video_completed
                    ↘ image_failed
image_completed → image_stale → image_generating
video_completed → video_stale → video_generating
```

| 状态 | 允许编辑 | 允许重生成图 | 允许生成视频 | 说明 |
| --- | --- | --- | --- | --- |
| `draft` | 是 | 否 | 否 | 分镜未完整 |
| `ready` | 是 | 是 | 否 | 可生成分镜图 |
| `image_generating` | 否 | 否 | 否 | 防止运行中输入变化 |
| `image_completed` | 是 | 是 | 是 | 图像可用 |
| `image_failed` | 是 | 是 | 否 | 需修复或重试 |
| `image_stale` | 是 | 是 | 可选，但需提示旧图过期 | 分镜文字或参考变更 |
| `video_generating` | 部分禁用 | 否 | 否 | 视频运行中 |
| `video_completed` | 是 | 是 | 是 | 视频可用 |
| `video_failed` | 是 | 是 | 是 | 展示修复建议 |

### 6.3 任务状态

```text
queued → running → succeeded
queued → canceled
running → cancel_requested → canceled / succeeded / failed
queued → failed
running → failed
```

规则：

- `queued` 可取消；
- `running` 只能请求取消，不保证立即停止；
- `succeeded` 和 `failed` 不可逆，只能创建新任务；
- 失败任务必须保留错误类型、错误信息和重试入口；
- 批量任务的父任务状态由子任务聚合得出。

## 8. AI 生成质量标准

P0 不只验收“生成了”，必须验收“生成可用”。

### 8.1 角色一致性

可用标准：

- 同一角色在同一项目内五官、发型、主要服装、体型保持一致；
- 角色性别、年龄段、气质不与角色设定冲突；
- 多角色同框时主要角色可区分。

不可用：

- 主角五官明显变成另一个人；
- 发型/服装与确认参考图大幅冲突；
- 角色数量错误且影响叙事。

### 8.2 场景一致性

可用标准：

- 同一场景的空间类型、色调、关键物件保持一致；
- 画面符合项目视觉风格；
- 不出现与时代/世界观明显冲突的元素。

### 8.3 分镜可读性

可用标准：

- 画面能表达分镜核心信息点；
- 镜头类型大致符合要求，如全景/中景/特写；
- 角色、动作、情绪与分镜描述不冲突。

不可用：

- 看不出该镜头要表达什么；
- 关键角色缺失；
- 情绪方向相反；
- 构图严重遮挡主体。

### 8.4 视频可用性

可用标准：

- 主体动作符合提示词大意；
- 无明显肢体畸变、穿模、主体闪烁；
- 镜头运动与提示词不冲突；
- 视频片段可作为后续剪辑素材。

不可用：

- 人物几乎不动且任务目标是动作；
- 动的是错误主体；
- 多人乱动导致叙事失败；
- 闪烁、崩坏、穿模严重。

### 8.5 风格一致性

可用标准：

- 校准图通过后，批量生成图保持同一画风；
- 色彩、线条、光影无明显断层；
- 不出现 2D、3D、皮克斯、写实等风格混杂。

## 9. 测试用例级验收

### TC-P0-001 创建项目并解析剧本

前置条件：

- 用户已登录；
- 用户有工作区项目创建权限；
- 用户有可用额度；
- 准备一份标准剧本文本。

操作步骤：

1. 进入项目列表；
2. 点击创建项目；
3. 填写项目名称；
4. 粘贴剧本文本；
5. 选择画幅和分辨率；
6. 点击创建并解析。

预期结果：

1. 创建按钮在必填项完整前禁用；
2. 点击后 1 秒内进入 `parsing`；
3. 页面展示解析中状态；
4. 解析成功后展示剧集结构；
5. 自动生成角色、场景、道具候选；
6. 项目进入 `assets_pending`；
7. 失败时停留当前项目，提供重试。

### TC-P0-002 确认公共资产

前置条件：

- 项目处于 `assets_pending`；
- 已有角色、场景、道具候选。

操作步骤：

1. 打开公共资产模块；
2. 编辑一个角色描述；
3. 上传或生成角色参考图；
4. 确认关键角色和主要场景；

预期结果：

1. 系统推荐关键角色和主要场景；
2. 用户可以手动调整关键资产标记；
3. 未确认关键角色/主要场景时，不允许进入可通过校准状态；
4. 确认后项目进入 `assets_reviewing`；
5. 下一步主 CTA 为拆解分镜。

### TC-P0-003 拆解分镜并完成风格校准

前置条件：

- 项目处于 `assets_reviewing`；
- 关键角色和主要场景已确认。

操作步骤：

1. 打开分镜模块；
2. 点击拆解分镜；
3. 查看生成的分镜列表；
4. 选择 3 个代表性分镜；
5. 点击生成校准图；
6. 点击通过校准。

预期结果：

1. 拆解中显示任务状态；
2. 拆解成功后分镜按序展示；
3. 单镜对白超过 40 字时出现建议拆分提示；
4. 未选择 3 个代表性分镜时，生成校准图按钮禁用；
5. 校准生成显示逐项进度；
6. 单张失败可单张重试；
7. 通过后项目进入 `calibration_passed`；
8. 批量生成分镜图入口解锁。

### TC-P0-004 批量生成分镜图支持部分成功

前置条件：

- 项目处于 `calibration_passed`；
- 校准已通过或已显式跳过；
- 额度充足；
- 存在至少 3 个 ready 分镜。

操作步骤：

1. 打开分镜模块；
2. 编辑其中一个分镜；
3. 点击批量生成分镜图；
4. 模拟部分分镜生成失败。

预期结果：

1. 批量生成时每个分镜显示独立状态；
2. 成功分镜展示生成图；
3. 失败分镜进入 `image_failed`；
4. 批量父任务展示部分失败；
5. 部分失败不影响其他分镜完成；
6. 失败分镜可筛选并重试。

### TC-P0-005 单分镜重生成保留版本

前置条件：

- 至少一个分镜图已完成。

操作步骤：

1. 打开分镜详情；
2. 修改画面描述；
3. 点击保存并重生成；
4. 等待生成完成；
5. 查看生成历史。

预期结果：

1. 修改后旧图标记为可能过期；
2. 重生成期间按钮禁用；
3. 生成成功后新图成为当前版本；
4. 旧版本仍可查看；
5. 生成失败时旧版本仍保留。

### TC-P0-006 分镜图转视频失败可恢复

前置条件：

- 分镜图已完成；
- 额度充足。

操作步骤：

1. 打开分镜详情；
2. 点击生成视频；
3. 模拟视频生成失败；
4. 查看失败提示；
5. 修改视频提示词后重试。

预期结果：

1. 视频任务进入 `video_generating`；
2. 失败后进入 `video_failed`；
3. 页面显示失败类型和修复建议；
4. 用户可在 3 步内重新发起任务；
5. 重试不覆盖原有分镜图。

### TC-P0-007 导出素材包

前置条件：

- 项目至少有一个完成分镜图；
- 用户有导出权限。

操作步骤：

1. 进入导出模块；
2. 点击导出素材包；
3. 等待导出完成；
4. 点击下载。

预期结果：

1. 导出前显示资产完整性检查；
2. 存在缺失资产时列出缺失项；
3. 用户可取消、修复或确认导出不完整包；
4. 导出成功后生成下载链接；
5. 导出失败允许重试。

### TC-P0-008 额度不足时阻止批量生成

前置条件：

- 项目处于 `calibration_passed`；
- 存在 10 个待生成分镜；
- 组织可用额度低于本次批量生成预估额度。

操作步骤：

1. 进入分镜模块；
2. 点击批量生成分镜图。

预期结果：

1. 系统在创建任务前完成额度校验；
2. 不创建批量任务和子任务；
3. 页面提示“额度不足，无法开始生成”；
4. 用户可减少生成范围或调整项目预算；
5. 分镜状态保持不变。

### TC-P0-009 未通过校准时批量生成禁用

前置条件：

- 项目已拆解分镜，状态为 `shots_ready` 或 `calibration_pending`；
- 校准未通过，也未显式跳过。

操作步骤：

1. 进入分镜模块；
2. 查看批量生成分镜图按钮；
3. 尝试触发批量生成。

预期结果：

1. 批量生成按钮禁用；
2. 页面显示阻塞原因：需要先完成风格校准；
3. 用户点击提示后进入风格校准模块；
4. 系统不创建生成任务。

### TC-P0-010 剧本解析失败后可重试

前置条件：

- 用户有项目编辑权限；
- 上传一份无法解析或模型返回异常的剧本。

操作步骤：

1. 创建项目并触发解析；
2. 等待解析失败；
3. 点击重试；
4. 替换为标准剧本后再次解析。

预期结果：

1. 首次失败后项目回到 `script_input` 或保持可编辑解析失败态；
2. 页面展示失败类型；
3. 用户可重试或替换剧本；
4. 重试成功后生成剧集结构和资产候选；
5. 不产生重复项目。

### TC-P0-011 批量生成中刷新页面后状态恢复

前置条件：

- 项目处于 `calibration_passed`；
- 已发起批量生成分镜图；
- 至少一个子任务仍在运行。

操作步骤：

1. 刷新浏览器页面；
2. 重新进入项目工作台；
3. 打开分镜模块。

预期结果：

1. 页面恢复当前批量任务状态；
2. 已完成分镜展示结果；
3. 运行中分镜展示生成中；
4. 失败分镜展示失败和重试入口；
5. 不重复创建批量任务。

### TC-P0-012 运行中重复点击不创建重复任务

前置条件：

- 单个分镜图正在生成中。

操作步骤：

1. 打开该分镜详情；
2. 观察重生成按钮；
3. 尝试重复点击或刷新后再次点击生成。

预期结果：

1. 运行中按钮禁用或显示“生成中”；
2. 系统不创建重复任务；
3. 原任务完成后按钮恢复；
4. 若任务失败，按钮变为可重试。

### TC-P0-013 无权限用户不能编辑或导出

前置条件：

- 用户只有项目只读权限；
- 项目已有分镜图资产。

操作步骤：

1. 打开项目；
2. 尝试编辑分镜；
3. 尝试导出素材包。

预期结果：

1. 用户可以查看有权限的数据；
2. 编辑入口禁用或隐藏；
3. 导出入口禁用或提示无权限；
4. 系统不创建编辑、生成或导出任务；
5. 不暴露项目敏感配置。

### TC-P0-014 导出时资产缺失被拦截

前置条件：

- 项目有分镜列表；
- 部分分镜图缺失或生成失败。

操作步骤：

2. 点击导出素材包。

预期结果：

1. 系统列出缺失或失败资产；
2. 用户可取消导出；
3. 用户可跳转修复缺失项；
4. 用户可二次确认导出不完整包；
5. 未确认前不开始导出任务。
